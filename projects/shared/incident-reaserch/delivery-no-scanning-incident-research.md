# Дослідження інцидентів: Немає сканування секретів, залежностей і образів

## Контекст з матриці

- Домен: CI/CD, репозиторії та доставка змін
- Зона: Перевірки коду та образів
- Загроза: Немає сканування секретів, залежностей і образів
- Ризик: Відсутність перевірок підвищує шанс не помітити секрет у коді, вразливу залежність або проблемний container image до того, як він буде використаний у delivery path.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Немає сканування секретів, залежностей і образів`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Середня` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Toyota T-Connect - access key exposed on GitHub](delivery-no-scanning-incident-research.md): Toyota T-Connect показує, що ключ доступу у GitHub може роками створювати шлях до дані клієнтів для ризику `Немає сканування секретів, залежностей і образів`. [Microsoft AI researchers - exposed Azure SAS token](delivery-no-scanning-incident-research.md): Microsoft AI SAS 2023 показує, що один надмірний хмарний токен спільного доступу може розкрити великий обсяг внутрішні дані для ризику `Немає сканування секретів, залежностей і образів`. [Uber 2016 - GitHub credentials and AWS S3 data](delivery-no-scanning-incident-research.md): Uber 2016 є прямим прикладом, як секрет у GitHub відкрив шлях до AWS S3 datastore для ризику `Немає сканування секретів, залежностей і образів`. [Codecov - compromised Bash Uploader](delivery-no-scanning-incident-research.md): Codecov 2021 демонструє, як скомпрометований допоміжний інструмент CI/CD може витягнути секрети із середовища конвеєра доставки для ризику `Немає сканування секретів, залежностей і образів`. [XZ Utils - backdoor CVE-2024-3094](delivery-no-scanning-incident-research.md): XZ Utils показує, що dependency без достатніх перевірок може внести бекдор у downstream середовища для ризику `Немає сканування секретів, залежностей і образів`.
