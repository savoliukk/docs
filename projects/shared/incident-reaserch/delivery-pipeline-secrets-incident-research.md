# Дослідження інцидентів: Викрадення секретів CI/CD

## Контекст з матриці

- Домен: CI/CD, репозиторії та доставка змін
- Зона: Jenkins / Argo CD / registry
- Загроза: Викрадення секретів CI/CD
- Ризик: Jenkins та інші елементи доставки коду можуть мати доступи до GitHub, registry, Kubernetes, хмари або середовищ. Викрадення цих секретів може відкрити кілька критичних систем одразу.
- Поточна пріоритетність: Висока

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Викрадення секретів CI/CD`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Викрадення секретів CI/CD' у зоні 'Jenkins / Argo CD / registry', бо показує реальний наслідок схожого розриву: Компрометація пристрою (endpoint) обійшла MFA через викрадення сесійного cookie і відкрила доступ до секретів CI/CD.

## Висновок

- Повторювані патерни: CI/CD-конвеєр виконував сторонній завантажувач із секретами в середовищі виконання. ; Компрометація пристрою (endpoint) обійшла MFA через викрадення сесійного cookie і відкрила доступ до секретів CI/CD. ; Збережені OAuth-токени й конфігураційні змінні конвеєра доставки створили широкий радіус ураження між платформою як сервісом (PaaS), GitHub і CI/CD..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Висока` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.

[CircleCI - malware on engineer laptop and stolen 2FA-backed session](delivery-pipeline-secrets-incident-research.md): CircleCI 2023 показує, що шкідливе ПЗ на пристрої (endpoint) може обійти MFA через сесійний токен і поставити під ризик секрети CI/CD для ризику `Викрадення секретів CI/CD`. [Heroku - stolen GitHub OAuth tokens and config vars](delivery-pipeline-secrets-incident-research.md): Heroku 2022 показує, що OAuth-токени та конфігураційні секрети можуть зв'язати платформу як сервіс (PaaS), GitHub і CI/CD в один шлях компрометації для ризику `Викрадення секретів CI/CD`. [Uber 2016 - GitHub credentials and AWS S3 data](delivery-pipeline-secrets-incident-research.md): Uber 2016 є прямим прикладом, як секрет у GitHub відкрив шлях до AWS S3 datastore для ризику `Викрадення секретів CI/CD`. [Microsoft AI researchers - exposed Azure SAS token](delivery-pipeline-secrets-incident-research.md): Microsoft AI SAS 2023 показує, що один надмірний хмарний токен спільного доступу може розкрити великий обсяг внутрішні дані для ризику `Викрадення секретів CI/CD`.
