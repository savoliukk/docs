# Дослідження інцидентів: Немає формалізованої ролі для ухвалення рішень

## Контекст з матриці

- Домен: Реагування на інциденти
- Зона: План реагування
- Загроза: Немає формалізованої ролі для ухвалення рішень
- Ризик: Якщо рішення приймає "команда" без формалізованої ролі, під час інциденту може бути затримка з ізоляцією систем, відкликанням доступів або комунікацією.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Немає формалізованої ролі для ухвалення рішень`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

- Статус дедуплікації: повний первинний опис інциденту подано в [access-mfa](access-mfa-incident-research.md); тут наведено тільки стислий зв'язок із поточним ризиком.

- Тип джерела: regulatory complaint
- Джерела:
  - [FTC revised complaint](https://www.ftc.gov/system/files/documents/cases/1523054_uber_technologies_revised_complaint_0.pdf)
- Стислий опис: Uber 2016 є прямим прикладом, як секрет у GitHub відкрив шлях до AWS S3 datastore.
- Використаний розрив у контролях або ризиковий розрив у цьому рядку: Секрети зберігались у GitHub, а доступ до GitHub залежав від персональних акаунтів без достатньої MFA-політики.
- Наслідки для цього контексту: Інцидент зачепив десятки мільйонів користувачів і водіїв та призвів до регуляторних наслідків.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Немає формалізованої ролі для ухвалення рішень' у зоні 'План реагування', бо показує реальний наслідок схожого розриву: Секрети зберігались у GitHub, а доступ до GitHub залежав від персональних акаунтів без достатньої MFA-політики.
- Рівень доказовості: високий
- Чи підсилює пріоритетність ризику: так
- Згадки про інструменти безпеки, якщо є: GitHub, AWS S3
