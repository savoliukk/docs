# Дослідження інцидентів: Ключі зовнішніх провайдерів

## Контекст з матриці

- Домен: Хмарна інфраструктура та зовнішні провайдери
- Зона: Firebase / FCM / Telegram / SMSC / пошта
- Загроза: Ключі зовнішніх провайдерів
- Ризик: Ключі Firebase/FCM, Telegram токен, SMSC/email облікові дані можуть дати контроль над ідентичністю, push/SMS/email і комунікаціями підтримки. Можливі масові повідомлення, фішинг від імені сервісу або витік діалогів/файлів підтримки.
- Поточна пріоритетність: Висока

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Ключі зовнішніх провайдерів`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

## Висновок

- Повторювані патерни: Email provider/інструменти технічної підтримки став upstream шляхом до фішинг проти клієнтів клієнта. ; Secret scanning/repository controls не виявили ключ доступу у публічному репозиторії протягом років. ; Людські облікові записи та support-процеси стали шляхом до даних клієнтів..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Висока` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Twilio - smishing проти співробітників](cloud-provider-keys-incident-research.md): Twilio 2022 показує, що фішинг проти співробітників може відкрити доступ до клієнтського контексту провайдера комунікацій для ризику `Ключі зовнішніх провайдерів`. [Mailchimp - targeted incident against crypto customers](cloud-provider-keys-incident-research.md): Mailchimp 2022 показує, що компрометація акаунта або адмін-доступу провайдера може перетворитися на фішинг від імені клієнта для ризику `Ключі зовнішніх провайдерів`. [Toyota T-Connect - access key exposed on GitHub](cloud-provider-keys-incident-research.md): Toyota T-Connect показує, що ключ доступу у GitHub може роками створювати шлях до дані клієнтів для ризику `Ключі зовнішніх провайдерів`. [Microsoft AI researchers - exposed Azure SAS token](cloud-provider-keys-incident-research.md): Microsoft AI SAS 2023 показує, що один надмірний хмарний токен спільного доступу може розкрити великий обсяг внутрішні дані для ризику `Ключі зовнішніх провайдерів`. [Uber 2016 - GitHub credentials and AWS S3 data](cloud-provider-keys-incident-research.md): Uber 2016 є прямим прикладом, як секрет у GitHub відкрив шлях до AWS S3 datastore для ризику `Ключі зовнішніх провайдерів`.
