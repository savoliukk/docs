# Дослідження інцидентів: Немає ізольованих або незмінних резервних копій

## Контекст з матриці

- Домен: Резервні копії та відновлення
- Зона: Незмінні копії
- Загроза: Немає ізольованих або незмінних резервних копій
- Ризик: Якщо резервні копії доступні для зміни з тих самих середовищ, що й бойове середовище, зловмисник або ransomware може пошкодити і основні дані, і резервні копії.
- Поточна пріоритетність: Висока

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Немає ізольованих або незмінних резервних копій`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

## Висновок

- Повторювані патерни: Завдання резервного копіювання і збої моніторингу не гарантували відновлення в момент інциденту. ; Резервні копії або знімки, розміщені в тому самому домені відмови, не захищають від фізичної втрати майданчика. ; Площина резервного копіювання/керування був доступний з того самого привілейованого хмарного акаунта, що й виробничих артефактів..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Висока` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Code Spaces - AWS console compromise and backup deletion](backup-no-immutable-incident-research.md): Code Spaces 2014 показує, що резервні копії без ізоляції від площини керування можуть бути знищені разом із бойовим середовищем для ризику `Немає ізольованих або незмінних резервних копій`. [GitLab.com - database incident and failed backups](backup-no-immutable-incident-research.md): GitLab 2017 показує, що резервне копіювання без перевіреного відновлення може не спрацювати саме під час інциденту для ризику `Немає ізольованих або незмінних резервних копій`.
