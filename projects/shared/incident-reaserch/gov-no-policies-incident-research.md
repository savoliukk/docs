# Дослідження інцидентів: Немає базового набору політик

## Контекст з матриці

- Домен: Управління безпекою і робочі процеси
- Зона: Політики та перевірки
- Загроза: Немає базового набору політик
- Ризик: Відсутність політик контролю доступу, паролів, прийнятного використання, реагування на інциденти й резервного копіювання ускладнює застосування правил і створює різне розуміння того, що дозволено.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Немає базового набору політик`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

### 4. GitHub - витік internal repositories через poisoned VS Code extension

- Статус дедуплікації: повний опис актуального інциденту подано в [access-mfa](access-mfa-incident-research.md); тут наведено стислий зв'язок із поточним ризиком.
- Дата публічного розкриття, публікації або оновлення: 20-26 травня 2026
- Дата атаки або виявлення, якщо відома: GitHub виявив і стримав компрометацію 18 травня 2026
- Тип джерела: офіційний блог безпеки / reliable media
- Джерела:
  - [GitHub Blog](https://github.blog/security/investigating-unauthorized-access-to-githubs-internal-repositories/)
  - [TechCrunch](https://techcrunch.com/2026/05/20/github-says-hackers-stole-data-from-thousands-of-internal-repositories/)
- Стислий опис українською: GitHub 2026 показує, що шкідливий інструмент розробника на одному пристрої (endpoint) може відкрити внутрішні репозиторії і запустити ротацію критичних секретів.
- Використаний розрив у контролях або ризиковий розрив: Компрометований пристрій розробника (endpoint) з доступом до репозиторії з вихідним кодом і секрети створив шлях до внутрішнього коду та облікові дані та пов'язані секрети.
- Наслідки: За публічним описом, вплив обмежувався внутрішні репозиторії; GitHub не мав доказів впливу на клієнтські репозиторії outside внутрішні репозиторії, але виконував ротацію секретів і аналіз журналів.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс ілюструє актуальний прояв такого розриву у 2025/2026 для загрози 'Немає базового набору політик' у зоні 'Політики та перевірки', оскільки Компрометований пристрій розробника (endpoint) з доступом до репозиторії з вихідним кодом і секрети створив шлях до внутрішнього коду та облікові дані та пов'язані секрети.

## Висновок

- Повторювані патерни: Внутрішні інструменти адміністрування й людські акаунти не були достатньо ізольовані від сценарію соціальної інженерії. ; Патчинг, облік власників активів і моніторинг health не були достатньо керованими як процес. ; Секрети зберігались у GitHub, а доступ до GitHub залежав від персональних акаунтів без достатньої MFA-політики..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Середня` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Uber 2016 - GitHub credentials and AWS S3 data](gov-no-policies-incident-research.md): Uber 2016 є прямим прикладом, як секрет у GitHub відкрив шлях до AWS S3 datastore для ризику `Немає базового набору політик`. [Twitter - компрометація внутрішніх admin tools](gov-no-policies-incident-research.md): Twitter 2020 демонструє, як компрометація співробітників може перейти в контроль над внутрішніми інструментами адміністрування для ризику `Немає базового набору політик`. [Equifax - Apache Struts breach and monitoring gaps](gov-no-policies-incident-research.md): Equifax 2017 демонструє, що прогалини в обліку власників активів, оновленнях і моніторингу можуть разом приховати масовий злам/витік для ризику `Немає базового набору політик`. [Target - missed security alerts](gov-no-policies-incident-research.md): Target 2013 показує, що навіть наявні сповіщення безпеки не допомагають без відповідальної маршрутизації сповіщень і реагування для ризику `Немає базового набору політик`. [Morgan Stanley Smith Barney - improper disposal of devices](gov-no-policies-incident-research.md): Morgan Stanley 2022 показує, що неочищені або незашифровані пристрої можуть розкрити персональні дані клієнтів (персональні дані (PII)) після списання для ризику `Немає базового набору політик`.
