# Дослідження інцидентів: Невідомі привілеї сервісних акаунтів

## Контекст з матриці

- Домен: Kubernetes та контейнерне середовище
- Зона: RBAC і сервісні акаунти
- Загроза: Невідомі привілеї сервісних акаунтів
- Ризик: Без інвентаризації сервісних акаунтів і прив'язок ролей важко зрозуміти, які workload-и або автоматизація мають надмірні права. Компрометація pod-а може перейти в ширший доступ.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Невідомі привілеї сервісних акаунтів`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

- Статус дедуплікації: повний первинний опис інциденту подано в [prod-pm2](prod-pm2-incident-research.md); тут наведено тільки стислий зв'язок із поточним ризиком.
- Рік і дата інциденту або публікації: 2023
- Тип джерела: звіт постачальника про загрозу
- Джерела:
  - [Sysdig SCARLETEEL MITRE analysis](https://www.sysdig.com/blog/scarleteel-mitre-attack)
- Стислий опис: SCARLETEEL показує, як скомпрометоване робоче навантаження (workload) може перейти до облікові дані AWS й ширшого хмарного доступу.
- Використаний розрив у контролях або ризиковий розрив у цьому рядку: Workload/cloud boundary дозволив рух від container/Kubernetes контексту до хмарних облікових даних.
- Наслідки для цього контексту: Звіт описує крадіжку даних, crypto-mining і persistence/бічний рух атакувальника у cloud-native середовищах.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Невідомі привілеї сервісних акаунтів' у зоні 'RBAC і сервісні акаунти', бо показує реальний наслідок схожого розриву: Workload/cloud boundary дозволив рух від container/Kubernetes контексту до хмарних облікових даних.
- Рівень доказовості: середній
- Чи підсилює пріоритетність ризику: так
- Згадки про інструменти безпеки, якщо є: Kubernetes, AWS metadata, CloudTrail

### 4. CircleCI - malware on engineer laptop and stolen 2FA-backed session

- Статус дедуплікації: повний первинний опис інциденту подано в [secrets-chat](secrets-chat-incident-research.md); тут наведено тільки стислий зв'язок із поточним ризиком.

- Тип джерела: official incident report
- Джерела:
  - [CircleCI incident report](https://circleci.com/blog/jan-4-2023-incident-report/)
  - [CircleCI security alert](https://circleci.com/blog/january-4-2023-security-alert/)
- Стислий опис: CircleCI 2023 показує, що шкідливе ПЗ на пристрої (endpoint) може обійти MFA через сесійний токен і поставити під ризик секрети CI/CD.
- Використаний розрив у контролях або ризиковий розрив у цьому рядку: Компрометація пристрою (endpoint) обійшла MFA через викрадення сесійного cookie і відкрила доступ до секретів CI/CD.
- Наслідки для цього контексту: CircleCI закликала клієнтів rotate секрети, перевіряти журнали і вживала containment-дій.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Невідомі привілеї сервісних акаунтів' у зоні 'RBAC і сервісні акаунти', бо показує реальний наслідок схожого розриву: Компрометація пристрою (endpoint) обійшла MFA через викрадення сесійного cookie і відкрила доступ до секретів CI/CD.
- Рівень доказовості: високий
- Чи підсилює пріоритетність ризику: так
- Згадки про інструменти безпеки, якщо є: SSO, 2FA, секрети CI/CD, шкідливе ПЗ на пристрої (endpoint)

### 5. Heroku - stolen GitHub OAuth tokens and config vars

- Тип джерела: офіційний розбір інциденту
- Джерела:
  - [Heroku April 2022 Incident Review](https://www.heroku.com/blog/april-2022-incident-review/)

- Наслідки для цього контексту: Heroku відкликала токени, повідомляла клієнтів і ротувала доступи; потенційно постраждали приватні репозиторії й секрети.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Невідомі привілеї сервісних акаунтів' у зоні 'RBAC і сервісні акаунти', бо показує реальний наслідок схожого розриву: Збережені OAuth-токени й конфігураційні змінні конвеєра доставки створили широкий радіус ураження між платформою як сервісом (PaaS), GitHub і CI/CD.
- Рівень доказовості: високий

[Heroku - stolen GitHub OAuth tokens and config vars](k8s-serviceaccounts-incident-research.md): Heroku 2022 показує, що OAuth-токени та конфігураційні секрети можуть зв'язати платформу як сервіс (PaaS), GitHub і CI/CD в один шлях компрометації для ризику `Невідомі привілеї сервісних акаунтів`.
