# Дослідження інцидентів: Витік коду або чутливого контексту через AI-інструменти

## Контекст з матриці

- Домен: Документація, код і AI-контекст
- Зона: AI/LLM-контекст
- Загроза: Витік коду або чутливого контексту через AI-інструменти
- Ризик: У тезах окремо зазначений ризик витоку власного коду через несанкціоновані AI/LLM інструменти. Також можливий витік внутрішніх назв хостів, коротких інструкцій, правил іменування секретів або задач із безпеки у зовнішній контекст.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Витік коду або чутливого контексту через AI-інструменти`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

> Обмеження релевантності: Для AI/LLM leakage є менше офіційних postmortem, тому report включає тільки ті кейси, де є надійні публічні матеріали.

### 1. Samsung - confidential data pasted into ChatGPT

- Статус дедуплікації: повний первинний опис інциденту подано в [data-local-copy](data-local-copy-incident-research.md); тут наведено тільки стислий зв'язок із поточним ризиком.
- Рік і дата інциденту або публікації: 2023
- Тип джерела: reliable media
- Джерела:
  - [The Register coverage](https://www.theregister.com/2023/04/06/samsung_reportedly_leaked_its_own/)
  - [Tom's Hardware coverage](https://www.tomshardware.com/news/samsung-fab-workers-leak-confidential-data-to-chatgpt)
- Стислий опис: Samsung 2023 показує, що без правил використання AI/LLM співробітники можуть винести код і внутрішній контекст у зовнішній LLM.
- Використаний розрив у контролях або ризиковий розрив у цьому рядку: AI/LLM usage policy і data-handling guardrails не запобігли передачі sensitive context зовнішньому сервісу.
- Наслідки для цього контексту: Samsung обмежувала використання генеративного AI і розглядала внутрішні AI-інструменти.
- Релевантність до цієї загрози/ризику: Релевантність непряма або часткова: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Витік коду або чутливого контексту через AI-інструменти' у зоні 'AI/LLM-контекст', бо показує реальний наслідок схожого розриву: AI/LLM usage policy і data-handling guardrails не запобігли передачі sensitive context зовнішньому сервісу.
- Рівень доказовості: середній
- Чи підсилює пріоритетність ризику: частково
- Згадки про інструменти безпеки, якщо є: ChatGPT, внутрішня AI-політика

### 3. GitHub - витік internal repositories через poisoned VS Code extension

- Статус дедуплікації: повний опис актуального інциденту подано в [access-mfa](access-mfa-incident-research.md); тут наведено стислий зв'язок із поточним ризиком.

- Дата атаки або виявлення, якщо відома: GitHub виявив і стримав компрометацію 18 травня 2026
- Тип джерела: офіційний блог безпеки / reliable media
- Джерела:
  - [GitHub Blog](https://github.blog/security/investigating-unauthorized-access-to-githubs-internal-repositories/)
  - [TechCrunch](https://techcrunch.com/2026/05/20/github-says-hackers-stole-data-from-thousands-of-internal-repositories/)
- Стислий опис українською: GitHub 2026 показує, що шкідливий інструмент розробника на одному пристрої (endpoint) може відкрити внутрішні репозиторії і запустити ротацію критичних секретів.
- Використаний розрив у контролях або ризиковий розрив: Компрометований пристрій розробника (endpoint) з доступом до репозиторії з вихідним кодом і секрети створив шлях до внутрішнього коду та облікові дані та пов'язані секрети.

- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс ілюструє актуальний прояв такого розриву у 2025/2026 для загрози 'Витік коду або чутливого контексту через AI-інструменти' у зоні 'AI/LLM-контекст', оскільки Компрометований пристрій розробника (endpoint) з доступом до репозиторії з вихідним кодом і секрети створив шлях до внутрішнього коду та облікові дані та пов'язані секрети.

## Висновок

- Повторювані патерни: AI/LLM usage policy і data-handling guardrails не запобігли передачі sensitive context зовнішньому сервісу. ; Зовнішній обмін даними через токен мав надмірні права й довгий строк дії..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Середня` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Samsung - confidential data pasted into ChatGPT](docs-ai-leak-incident-research.md): Samsung 2023 показує, що без правил використання AI/LLM співробітники можуть винести код і внутрішній контекст у зовнішній LLM для ризику `Витік коду або чутливого контексту через AI-інструменти`.
