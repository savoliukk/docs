# Дослідження інцидентів: Неповний реєстр систем і власників

## Контекст з матриці

- Домен: Управління безпекою і робочі процеси
- Зона: Власники
- Загроза: Неповний реєстр систем і власників
- Ризик: Неповний SaaS-реєстр із власниками і неповний список внутрішніх сервісів ускладнюють розуміння, які системи потрібно захищати, хто за них відповідає і що відкликати при звільненні людини.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Неповний реєстр систем і власників`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

## Висновок

- Повторювані патерни: Площина резервного копіювання/керування був доступний з того самого привілейованого хмарного акаунта, що й виробничих артефактів. ; Внутрішня база support analytics була помилково доступна з інтернету. ; Патчинг, облік власників активів і моніторинг health не були достатньо керованими як процес..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Середня` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Equifax - Apache Struts breach and monitoring gaps](gov-no-system-register-incident-research.md): Equifax 2017 демонструє, що прогалини в обліку власників активів, оновленнях і моніторингу можуть разом приховати масовий злам/витік для ризику `Неповний реєстр систем і власників`. [Microsoft - exposed customer support database](gov-no-system-register-incident-research.md): Microsoft 2020 показує, що помилково налаштована база технічної підтримки може розкрити клієнтський контекст без окремої атаки на застосунок для ризику `Неповний реєстр систем і власників`. [Code Spaces - AWS console compromise and backup deletion](gov-no-system-register-incident-research.md): Code Spaces 2014 показує, що резервні копії без ізоляції від площини керування можуть бути знищені разом із бойовим середовищем для ризику `Неповний реєстр систем і власників`. [OVHcloud Strasbourg - data center fire and backup locality](gov-no-system-register-incident-research.md): OVHcloud Strasbourg показує, що резервне копіювання у тому самому домені відмови може не пережити фізичний інцидент для ризику `Неповний реєстр систем і власників`. [Capital One - cloud data breach](gov-no-system-register-incident-research.md): Capital One 2019 показує, як розрив у Cloud IAM може відкрити доступ до масиву критичних даних для ризику `Неповний реєстр систем і власників`.
