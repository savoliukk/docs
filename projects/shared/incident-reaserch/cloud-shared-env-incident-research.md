# Дослідження інцидентів: Бойове і небойові середовища не розділені повністю

## Контекст з матриці

- Домен: Хмарна інфраструктура та зовнішні провайдери
- Зона: AWS
- Загроза: Бойове і небойові середовища не розділені повністю
- Ризик: Часткове розділення dev/stage/prod і відсутність окремих хмарного акаунтаs/projects для prod та non-prod може дозволити помилці або компрометації небойового середовища вплинути на бойове.
- Поточна пріоритетність: Середня

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Бойове і небойові середовища не розділені повністю`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

## Висновок

- Повторювані патерни: Завдання резервного копіювання і збої моніторингу не гарантували відновлення в момент інциденту. ; Площина резервного копіювання/керування був доступний з того самого привілейованого хмарного акаунта, що й виробничих артефактів. ; Внутрішня база support analytics була помилково доступна з інтернету..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Середня` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
- Короткий текст для колонки "Пов'язані інциденти": [Code Spaces - AWS console compromise and backup deletion](cloud-shared-env-incident-research.md): Code Spaces 2014 показує, що резервні копії без ізоляції від площини керування можуть бути знищені разом із бойовим середовищем для ризику `Бойове і небойові середовища не розділені повністю`. [Microsoft - exposed customer support database](cloud-shared-env-incident-research.md): Microsoft 2020 показує, що помилково налаштована база технічної підтримки може розкрити клієнтський контекст без окремої атаки на застосунок для ризику `Бойове і небойові середовища не розділені повністю`. [GitLab.com - database incident and failed backups](cloud-shared-env-incident-research.md): GitLab 2017 показує, що резервне копіювання без перевіреного відновлення може не спрацювати саме під час інциденту для ризику `Бойове і небойові середовища не розділені повністю`. [OVHcloud Strasbourg - data center fire and backup locality](cloud-shared-env-incident-research.md): OVHcloud Strasbourg показує, що резервне копіювання у тому самому домені відмови може не пережити фізичний інцидент для ризику `Бойове і небойові середовища не розділені повністю`. [Capital One - cloud data breach](cloud-shared-env-incident-research.md): Capital One 2019 показує, як розрив у Cloud IAM може відкрити доступ до масиву критичних даних для ризику `Бойове і небойові середовища не розділені повністю`.
