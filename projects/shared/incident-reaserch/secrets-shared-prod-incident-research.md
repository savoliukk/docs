# Дослідження інцидентів: Спільний пароль до бойове середовище, серверів або адмін-панелей

## Контекст з матриці

- Домен: Паролі, секрети та криптографічні ключі
- Зона: Паролі людей
- Загроза: Спільний пароль до бойове середовище, серверів або адмін-панелей
- Ризик: Один пароль, відомий кільком людям, може дати доступ до критичних систем. Часткова ротація після звільнення залишає ризик повторного використання старого доступу.
- Поточна пріоритетність: Висока

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Спільний пароль до бойове середовище, серверів або адмін-панелей`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

- Потенційний або підтверджений подальший рух атакувальника: Компрометований action міг розкрити секрети у робочий процес журнали, створюючи шлях до downstream cloud, package registry або deployment облікові дані.
- Наслідки: GitHub тимчасово прибрав Action, CISA опублікувала alert, постраждалі організації мали перевіряти журнали і ротувати розкриті облікові дані.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс ілюструє актуальний прояв такого розриву у 2025/2026 для загрози 'Спільний пароль до бойове середовище, серверів або адмін-панелей' у зоні 'Паролі людей', оскільки Робочий процес використовував сторонній action за змінювані теги і запускав його в CI середовищі, де були секрети.
- Рівень доказовості: високий
- Чи підсилює пріоритетність ризику: так
- Згадки про інструменти безпеки, якщо є: GitHub Actions, tj-actions/changed-files, reviewdog/action-setup, секрети CI/CD


## Висновок

- Повторювані патерни: Площина резервного копіювання/керування був доступний з того самого привілейованого хмарного акаунта, що й виробничих артефактів. ; Remote access і парольна дисципліна були слабкими для системи, що керує фізичним процесом. ; VPN-доступ до корпоративного контуру не був достатньо захищений MFA та життєвим циклом акаунтів..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Висока` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.
