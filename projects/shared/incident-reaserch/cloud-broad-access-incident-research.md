# Дослідження інцидентів: Широкі хмарні доступи

## Контекст з матриці

- Домен: Хмарна інфраструктура та зовнішні провайдери
- Зона: AWS
- Загроза: Широкі хмарні доступи
- Ризик: У джерелі зафіксована гібридна IAM-модель: частина доступів побудована за принципом мінімальних потрібних прав, частина має широкі права. Компрометація такого акаунта може дати надмірний контроль над хмарними ресурсами.
- Поточна пріоритетність: Висока

## Пошуковий фокус

Шукались реальні публічні інциденти, де підтверджено або чітко описано схожий розрив у контролях чи ризиковий сценарій для теми: `Широкі хмарні доступи`. Пріоритет надавався офіційним післяінцидентним розборам, урядовим попередженням, регуляторним і судовим документам, звітам постачальників безпекових досліджень і надійним медіа. Усі описи нижче подані українською мовою; назви джерел, компаній, продуктів і технічних ідентифікаторів залишені в оригіналі.

## Інциденти

### 1. Capital One - cloud data breach

- Статус дедуплікації: повний первинний опис інциденту подано в [k8s-rbac](k8s-rbac-incident-research.md); тут наведено тільки стислий зв'язок із поточним ризиком.
- Рік і дата інциденту або публікації: 2019
- Тип джерела: government / court record
- Джерела:
  - [DOJ conviction announcement](https://www.justice.gov/usao-wdwa/pr/seattle-tech-worker-convicted-wire-fraud-and-computer-intrusions-capital-one-data)
- Стислий опис: Capital One 2019 показує, як розрив у Cloud IAM може відкрити доступ до масиву критичних даних.
- Використаний розрив у контролях або ризиковий розрив у цьому рядку: Cloud IAM/metadata/service boundary дозволив отримати доступ до великих обсягів даних.
- Наслідки для цього контексту: Capital One disclosed data breach для понад 100 млн заявок/клієнтів; інцидент мав регуляторні та судові наслідки.
- Релевантність до цієї загрози/ризику: Релевантність пряма: кейс підтверджує або ілюструє розрив у контролях або ризиковий розрив для загрози 'Широкі хмарні доступи' у зоні 'AWS', бо показує реальний наслідок схожого розриву: Cloud IAM/metadata/service boundary дозволив отримати доступ до великих обсягів даних.
- Рівень доказовості: високий
- Чи підсилює пріоритетність ризику: так
- Згадки про інструменти безпеки, якщо є: AWS, IAM role, S3

## Висновок

- Повторювані патерни: Площина резервного копіювання/керування був доступний з того самого привілейованого хмарного акаунта, що й виробничих артефактів. ; Cloud IAM/metadata/service boundary дозволив отримати доступ до великих обсягів даних. ; Kubernetes/control-plane інтерфейс був доступний без належної автентифікації..
- Наскільки поширений тип проблеми: знайдені кейси показують, що цей клас розриву трапляється в реальних інцидентах із SaaS-сервісами, хмарними середовищами, CI/CD, пристроями або процесами управління безпекою; точна поширеність не оцінювалась кількісно.
- Чи варто залишити поточну пріоритетність: поточну пріоритетність `Висока` варто залишити, якщо внутрішній стан контролів відповідає описаному ризику; для непрямих кейсів це слід трактувати як підтримувальний, а не абсолютний доказ.

[Code Spaces - AWS console compromise and backup deletion](cloud-broad-access-incident-research.md): Code Spaces 2014 показує, що резервні копії без ізоляції від площини керування можуть бути знищені разом із бойовим середовищем для ризику `Широкі хмарні доступи`. [Tesla - exposed Kubernetes console and cryptomining](cloud-broad-access-incident-research.md): Tesla 2018 показує, що розкрита консоль Kubernetes може стати шляхом до хмарних облікових даних й зловживання ресурсів для ризику `Широкі хмарні доступи`. [Imperva - Cloud WAF data exposure](cloud-broad-access-incident-research.md): Imperva 2019 показує, що cloud provider/service keys у витоку можуть створити downstream-ризик для клієнтів для ризику `Широкі хмарні доступи`. [Microsoft AI researchers - exposed Azure SAS token](cloud-broad-access-incident-research.md): Microsoft AI SAS 2023 показує, що один надмірний хмарний токен спільного доступу може розкрити великий обсяг внутрішні дані для ризику `Широкі хмарні доступи`.
